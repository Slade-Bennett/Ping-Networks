# Contributing to Ping-Networks

First off, thank you for considering contributing to Ping-Networks! We welcome any help, whether it's reporting a bug, suggesting a new feature, or submitting a pull request.

## How to Contribute

We follow a standard GitHub workflow for contributions. If you're new to this, here's a quick overview:

1.  **Fork the repository:** Click the "Fork" button on the top right of the repository page. This will create a copy of the repository in your own GitHub account.
2.  **Clone your fork:** Clone the forked repository to your local machine.
3.  **Create a new branch:** Create a new branch for your changes.
4.  **Make your changes:** Make your changes to the code, documentation, or whatever you're contributing.
5.  **Commit your changes:** Commit your changes with a clear and descriptive commit message.
6.  **Push your changes:** Push your changes to your forked repository.
7.  **Create a pull request:** Open a pull request from your forked repository to the main Ping-Networks repository.

## Style Guide

To maintain a consistent style throughout the project, please follow these guidelines:

*   **PowerShell:** Follow the community-accepted [PSScriptAnalyzer](https://github.com/PowerShell/PSScriptAnalyzer) style guide.
*   **Comments:** Add comments to your code where necessary, especially for complex logic.
*   **Commit Messages:** Write clear and concise commit messages.

## Reporting Bugs

If you find a bug, please open an issue on the GitHub repository. When reporting a bug, please include the following information:

*   **A clear and descriptive title:** For example, "Error when reading Excel file with no data."
*   **A detailed description of the bug:** Include the steps to reproduce the bug, the expected behavior, and the actual behavior.
*   **Your environment:** Include your PowerShell version, operating system, and any other relevant information.
*   **Any error messages or logs:** If you have any error messages or logs, please include them in the issue.

## Suggesting Enhancements

If you have an idea for a new feature or an enhancement to an existing feature, please open an issue on the GitHub repository. When suggesting an enhancement, please include the following information:

*   **A clear and descriptive title:** For example, "Add support for JSON output."
*   **A detailed description of the enhancement:** Explain the feature and why you think it would be a good addition to the project.
*   **Any relevant examples or mockups:** If you have any examples or mockups of how the feature would work, please include them in the issue.

Thank you for your contributions!
